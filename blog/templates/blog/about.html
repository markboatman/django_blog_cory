Django Blog App

This Django project is running on heroku and is using a postreSQL database for django data support as well as app specific data storage. App specific data includes user profiles and user blog posts. The project includes the main project app, a 'blog' app, and a 'user' app.

The project uses much of the valuable functionality Django provides. Users are instances of 'auth.models.User'. Users have a reference to a 'user.profile' object. Profile objects inherit from 'django.db.models.Model''. Users are created using a UserRegistrationForm that is a decendant of a Django UserCreationForm. User and Profile updates are handled by decendants of Django 'forms.ModelForm'.

User authentication and passsword recovery take advantage of the django.contrib.auth.views: LoginView, LogoutView, PasswordResetView, etc.

Profile image files are stored in an AWS S3 bucket. This is made possible through the use of the django-storage and boto3 packages.

Much of the heroku deployment configuration requirments are handled by the package django_heroku imported in the project's settings.py file.
